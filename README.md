# Análisis de Telemetría y Conectividad a Internet en Colombia

## 📝 Descripción del Proyecto
Este proyecto consiste en una herramienta automatizada desarrollada en **Python** para la recopilación, procesamiento analítico y visualización de datos estadísticos sobre el acceso a Internet en Colombia. El objetivo principal es estructurar grandes volúmenes de información pública, identificar patrones de conectividad y generar representaciones gráficas operativas que faciliten la toma de decisiones estratégicas y la comprensión del panorama de red nacional.

Esta metodología de procesamiento y visualización comparte los mismos principios fundamentales utilizados en el **Blue Team** para el análisis de tráfico masivo de red, la detección de anomalías en líneas de base operativas (*baselines*) y la generación de reportes ejecutivos.

## 🚀 Características Clave
* **Recopilación e Ingesta:** Automatización de la extracción y filtrado de fuentes de datos.
* **Procesamiento y Análisis:** Normalización y estructuración de la información utilizando librerías nativas de análisis de datos en Python.
* **Visualización Avanzada:** Generación de gráficos analíticos interactivos y estáticos para la identificación inmediata de tendencias y desviaciones en los datos.

## 🛠️ Tecnologías y Librerías Utilizadas
* **Lenguaje:** Python 3
* **Manipulación y Limpieza de Datos:** Pandas / NumPy

## 📂 Estructura del Repositorio
* `proyecto.py`: Script principal encargado de la lógica de análisis masivo y renderizado de gráficas.
* `limpieza.py`: Script secundario enfocado en la depuración, filtrado y normalización del set de datos original.
* `Data definitiva proyecto.csv`: Dataset estructurado con las métricas de conectividad analizadas.

## 🎯 Aplicación en Ciberseguridad (SecOps)
Las competencias técnicas aplicadas en este desarrollo demuestran la capacidad de:
1. Tratar y depurar registros masivos de información (similar al procesamiento de logs SIEM/Syslog).
2. Establecer comportamientos normales del entorno para la posterior detección de anomalías de red o transferencias inusuales de datos.
3. Comunicar métricas complejas a través de dashboards y reportes visuales claros para equipos de respuesta a incidentes.
